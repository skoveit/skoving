# Getting Started

The fastest and most reliable way to get SkoveNet running is to use our pre-compiled binaries. This guide will take you from zero to executing your first command (`ls`) across a local mesh.

## 1. Installation

The **best way** to install SkoveNet is to download the latest stable binaries from our [GitHub Releases](https://github.com/skoveit/skovenet/releases) page.

Download the `controller` and `sgen` binaries for your operator machine (Linux, Windows, or macOS).

---

## 2. Quick Start

Follow these steps to form a mesh and run a command on a local target.

### Step A: Generate the Agent
Use `sgen` to create a standalone agent for your target platform.
```bash
./sgen generate --os linux --arch amd64 --output agent
chmod +x agent
```

### Step B: Launch the Agent
Execute the agent on the target machine (or a separate terminal).
```bash
./agent
```
*The agent will generate a Peer ID and start listening for connections.*

### Step C: Launch the Controller
In a new terminal, start the operator interface.
```bash
./controller
```

### Step D: Execute `ls`
Inside the controller, perform the following sequence:
```bash
> sign <your_private_key>    # Authenticate (generated by sgen keygen)
> peers                      # Discover the local agent
> use <peerID>               # Select the agent
[peerID]> ls                 # Execute the command!
```
